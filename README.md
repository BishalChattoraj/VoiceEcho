# VoiceEcho — AI Voice Mood Analytics Platform

VoiceEcho is a modern web application designed to help users track their emotional wellness through text and voice journaling. Using AI-powered sentiment analysis and personalized advice, it provides insights into mood patterns and helps identify potential burnout.

## 🌟 Key Features

- **Multi-Modal Journaling**: Submit entries via text or audio recordings.
- **AI Sentiment Analysis**: Deep analysis of your mood using NLP.
- **Micro-Visualizations**: Real-time audio visualizers during recording and mood gauges for results.
- **Burnout Detection**: Automatically identifies consecutive negative mood patterns.
- **AI Wellness Advice**: Personalized coping strategies and reflections generated by Google Gemini.
- **Trend Analytics**: Comprehensive charts and graphs showing your emotional journey over weeks and months.
- **Secure Authentication**: JWT-based secure login and registration.

## 🛠 Tech Stack

### Frontend
- **HTML5/CSS3**: Custom design system with glassmorphism and smooth animations.
- **JavaScript (ESM)**: Vanilla JS for maximum performance.
- **Chart.js**: Dynamic data visualizations.
- **Web Audio API**: Real-time audio processing and visualization.

### Backend
- **Node.js & Express**: High-performance API architecture.
- **MongoDB & Mongoose**: Scalable NoSQL database for journal entries and user data.
- **Google Gemini (GenAI)**: Advanced AI for supportive wellness advice.
- **OpenAI Whisper**: State-of-the-art audio transcription.
- **Sentiment**: NLP-based sentiment score calculation.
- **Multer**: Secure multi-part file handling for audio uploads.

## 🚀 Getting Started

### Prerequisites
- Node.js (v18+)
- MongoDB (Local or Atlas)
- API Keys: 
  - `GEMINI_API_KEY` (Google AI Studio)
  - `OPENAI_API_KEY` (OpenAI Platform)

### Installation

1. **Clone the repository**
   ```bash
   git clone <repository-url>
   cd VoiceEcho
   ```

2. **Install dependencies**
   ```bash
   npm install
   ```

3. **Configure Environment Variables**
   Create a `.env` file in the root directory:
   ```env
   PORT=3000
   MONGODB_URI=your_mongodb_connection_string
   JWT_SECRET=your_jwt_secret
   GEMINI_API_KEY=your_google_gemini_key
   OPENAI_API_KEY=your_openai_whisper_key
   NODE_ENV=development
   ```

4. **Run the application**
   ```bash
   # Development mode with nodemon
   npm run dev

   # Production mode
   npm start
   ```

## 🐳 Docker Usage

VoiceEcho is fully containerized and can be run easily using Docker and Docker Compose.

### Using Docker Compose (Recommended)
This will set up both the application and a MongoDB instance automatically.

1. Ensure your `.env` file is configured with the necessary API keys (`GEMINI_API_KEY`, `OPENAI_API_KEY`).
2. Run the following command:
   ```bash
   docker compose up --build -d
   ```
3. Access the application at `http://localhost:3000`.

### Using Docker Directly
If you already have a MongoDB instance running:

1. Build the image:
   ```bash
   docker build -t voiceecho .
   ```
2. Run the container:
   ```bash
   docker run -p 3000:3000 --env-file .env voiceecho
   ```

## 📁 Project Structure

- `/public`: Frontend assets (HTML, CSS, JS).
- `/src`: Backend core (server setup, app configuration).
- `/routes`: API route definitions.
- `/controllers`: Request handling logic.
- `/models`: Mongoose database schemas.
- `/services`: Third-party integrations (AI, Transcription, Sentiment).
- `/uploads`: Temporary storage for audio recordings.
